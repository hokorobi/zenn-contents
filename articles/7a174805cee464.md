fenrir を使用する際、C-n / C-p で候補選択を行いたかったため、AutoHotkey（v2）で以下のような定義をしていました。

```autohotkey
^n::Send("{Down}")
^p::Send("{Up}")
```

しかし、MS-IMEの設定でも変換候補の選択に C-n / C-p を割り当てていたところ、問題が発生しました。
MS-IME での変換候補選択中、キー操作が MS-IME を突き抜けて fenrir 側にも二重に送られてしまうようです。
fenrir 側に候補が表示されていない状態だと、C-n は右移動、C-p は左移動として挙動するため、結果として MS-IMEの変換対象文字が左右に移動してしまう という現象が起きていました。

そこで、MS-IMEがオンの状態の場合は、fenrir内であっても C-n / C-p のキーバインドをバイパス（無効化）する ようにスクリプトを変更しました。

```autohotkey
#HotIf WinActive("ahk_exe fenrir.exe")
^n::C_n()
^p::C_p()
#HotIf
C_n() {
	if (IME_GET()) {
		Send(A_ThisHotkey)
	} else {
		Send("{Down}")
	}
}
C_p() {
	if (IME_GET()) {
		Send(A_ThisHotkey)
	} else {
		Send("{Up}")
	}
}
```

これで、MS-IME 変換時の予期せぬカーソル移動に煩わされることがなくなりました。
